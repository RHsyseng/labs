To have a full reference there’s a deploy script inside the deploy folder mentioned before, with all the steps shown here.

Create the pull-secret

This pull-secret will be injected into the discovery ISO, so it needs to be a valid pull-secret and be able to pull from the registries you’re going to use for the deployment. If you have some doubts you could use the next tool to verify it: pull secret validator

apiVersion: v1
kind: Secret
metadata:
  name: assisted-deployment-pull-secret
  namespace: open-cluster-management
stringData:
  .dockerconfigjson: "PULL_SECRET"
  type: kubernetes.io/dockerconfigjson

SSH private key

This SSH key will be used for the user core, the one we will be able to use to ssh into our CoreOS-deployed systems.

apiVersion: v1
kind: Secret
metadata:
  creationTimestamp: null
  name: assisted-deployment-ssh-private-key
  namespace: open-cluster-management
type: Opaque
stringData: --redacted--

ClusterImageSet

This is the reference we’ve used in the AI cluster deployment. So, you have to use the same name, and also replace the tag with the OCP release you’re using. If you want to query available image release look at this here

apiVersion: hive.openshift.io/v1
kind: ClusterImageSet
metadata:
  name: openshift-v4.8.0
  namespace: open-cluster-management
spec:
  releaseImage: TAG_OCP_IMAGE_RELEASE

Config map for the assisted service config

This is the way to pass some configuration to the assisted service. In our case just log level set to debug

apiVersion: v1
kind: ConfigMap
metadata:
  name: assisted-service-config
  namespace: open-cluster-management
  labels:
    app: assisted-service
data:
  LOG_LEVEL: "debug"

Agent Service Config

This file should contain the osImages which will be used.

 Important

Here if you’re using a Connected environment could be directly the mirror. But, If you’re using a disconnected environment, you need to download to the httpd server in order to be available:

 Disconnected

if you’re deploying with a lab kcli IPI plan bare-metal of this repository, in the script 05_cache.sh is there a section to download these images to the httpd server in the installer node.

Example YAML file:

---
apiVersion: agent-install.openshift.io/v1beta1
kind: AgentServiceConfig
metadata:
  namespace: open-cluster-management
  name: agent
  annotations:
    unsupported.agent-install.openshift.io/assisted-service-configmap: "assisted-service-config"
spec:
  databaseStorage:
    accessModes:
      - ReadWriteOnce
    resources:
      requests:
        storage: 40Gi
  filesystemStorage:
    accessModes:
      - ReadWriteOnce
    resources:
      requests:
        storage: 40Gi
  mirrorRegistryRef:
    name: "lab-index-mirror"
  osImages:
    - openshiftVersion: "4.8"
      version: ""
      url: "http://INSTALLER_IP/rhcos-live.x86_64.iso"
      rootFSUrl: "http://INSTALLER_IP/rhcos-live-rootfs.x86_64.img"

Mirror Config Map (only for disconnected environment)

This is a step required for the disconnected environment to get the information about the CA certificate, the bundle and the registries.conf info:

  • CA certificate:
cabundle=$(ssh -q root@$ip -x 'cat /opt/registry/certs/domain.crt')
echo -e "  ca-bundle.crt: |\n$(echo -n "$cabundle"  | sed "s/^/    /")" >> 05_Mirror_ConfigMap.yml

where the $ip is the variable for the installer helper node but depending of ipv4 or ipv6 will be a different formatted ip.

  • The registries.conf information
reg=$(ssh -q core@$ip0 -x 'cat /etc/containers/registries.conf')
echo -e "  registries.conf: |\n$(echo -n "$reg"  | sed "s/^/    /")" >> 05_Mirror_ConfigMap.yml

where the $ip is the variable for the installer helper node but depending of ipv4 or ipv6 will be a different formatted ip.

Create the cluster deployment with the new spec

The important thing here is the “cluster installer reference” just to point to the new configuration “agent Cluster Install” which will be defined in the next step. Also, remember that all the reference will be the new spoke name or namespace.

apiVersion: hive.openshift.io/v1
kind: ClusterDeployment
metadata:
  name: OC_SPOKE_NAME
  namespace: OC_SPOKE_NAME
spec:
  baseDomain: alklabs.com
  clusterInstallRef:
    group: extensions.hive.openshift.io
    kind: AgentClusterInstall
    name: OC_SPOKE_NAME
    version: v1beta1
  clusterName: OC_SPOKE_NAME
  controlPlaneConfig:
    servingCertificates: {}
  platform:
    agentBareMetal:
      agentSelector:
        matchLabels:
          cluster-name: "OC_SPOKE_NAME"
  pullSecretRef:
    name: assisted-deployment-pull-secret

Create new spoke namespace

The spoke will be deployed in a new namespace, you have to create it before continuing with the rest of manifests:

oc create project <OC_SPOKE_NAME>

Create the Agent Cluster Install

Note:

  • the controlPlaneAgents will be 1/3, and workerAgents will be xxx.
  • the sshPublicKey needs to be modified to include the public key pair.

So here we have two possibilities:

  • SNO: you have to set "" the api-vip and ingress-vip, but also you have to keep the machineNetwork
  • Multi-node: you have to set properly the api-vip and the ingress-vip, but you could omit the machineNetwork because it is auto-discovered from the two IPs

Another consideration is the imageSetRef which should be the same you created before as well as the clusterDeployment reference. Remember the name for the spoke

apiVersion: extensions.hive.openshift.io/v1beta1
kind: AgentClusterInstall
metadata:
  name: OC_SPOKE_NAME
  namespace: OC_SPOKE_NAME
spec:
  apiVIP: 2620:52:0:1304::43
  ingressVIP: 2620:52:0:1304::42
  clusterDeploymentRef:
    name: OC_SPOKE_NAME
  imageSetRef:
    name: openshift-v4.8.0
  networking:
    clusterNetwork:
      - cidr: "fd01::/48"
        hostPrefix: 64
    serviceNetwork:
      - "fd02::/112"
    machineNetwork:
      - cidr: "2620:52:0:1304::/64"
  provisionRequirements:
    controlPlaneAgents: NUMBER_OF_MASTERS
  sshPublicKey: PUB_SSH_KEY

Create the rest of manifests required by ACM

apiVersion: agent.open-cluster-management.io/v1
kind: KlusterletAddonConfig
metadata:
  name: OC_SPOKE_NAME
  namespace: OC_SPOKE_NAME
spec:
  clusterName: OC_SPOKE_NAME
  clusterNamespace: OC_SPOKE_NAME
  clusterLabels:
    cloud: auto-detect
    vendor: auto-detect
  workManager:
    enabled: true
  applicationManager:
    enabled: false
  certPolicyController:
    enabled: false
  iamPolicyController:
    enabled: false
  policyController:
    enabled: false
  searchCollector:
    enabled: false

apiVersion: cluster.open-cluster-management.io/v1
kind: ManagedCluster
metadata:
  name: OC_SPOKE_NAME
  namespace: OC_SPOKE_NAME
spec:
  hubAcceptsClient: true

apiVersion: v1
kind: Secret
metadata:
  name: assisted-deployment-pull-secret
  namespace: OC_SPOKE_NAME
stringData:
  .dockerconfigjson: "PULL_SECRET"
  type: kubernetes.io/dockerconfigjson

Create the InfraEnv

  • the sshPublicKey needs to be modified to include the public key pair.
  • the sshAuthorizedKey should be singular (not array)
  • IgnitionConfigOverride should be present
apiVersion: agent-install.openshift.io/v1beta1
kind: InfraEnv
metadata:
  name: OC_SPOKE_NAME
  namespace: OC_SPOKE_NAME
spec:
  clusterRef:
    name: OC_SPOKE_NAME
    namespace: OC_SPOKE_NAME
  sshAuthorizedKey: PUB_SSH_KEY
  agentLabelSelector:
    matchLabels:
      cluster-name: "OC_SPOKE_NAME"
  pullSecretRef:
    name: assisted-deployment-pull-secret
  ignitionConfigOverride: ""
