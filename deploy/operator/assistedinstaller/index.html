Assisted Installer Operator

Catalog Source

First thing is to create the catalog source with the quay image from the Development team.

The only thing we need to change is the tag for the image using the reference url

---
apiVersion: operators.coreos.com/v1alpha1
kind: CatalogSource
metadata:
  name: assisted-service
  namespace: openshift-marketplace
spec:
  image: quay.io/ocpmetal/assisted-service-index:<tag from quay.io>
  sourceType: grpc
  • In case you want to go with the version of the Operator Hub, in the extra folder you’ve got the file assisted-installer-CS-operatorHub.yml
  • In the case you want to compile your own image, you could follow the steps in the extra folder script: create-quay-image.sh

Requirements for Assisted Installer

The idea is to create the namespace and fulfill the remaining requirements before installing the operator:

---
apiVersion: v1
kind: Namespace
metadata:
  name: assisted-installer
  labels:
    name: assisted-installer

AI Operator Group

---
apiVersion: operators.coreos.com/v1
kind: OperatorGroup
metadata:
  name: assisted-installer-operator
  namespace: assisted-installer
spec:
  targetNamespaces:
    - assisted-installer

Subscription AI

Now it’s time for the subscription, but there are important things to keep in mind:

  • OpenShift version that you have to force deploying the 4.8
  • Deploy_target: onprem if you’re on BareMetalHost
---
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: assisted-service-operator
  namespace: assisted-installer
spec:
  channel: alpha
  installPlanApproval: Automatic
  name: assisted-service-operator
  source: assisted-service
  sourceNamespace: openshift-marketplace

At this point, now this should be fixed, but we will keep it here, in case you have any problem with the HTTPS certificate validation using the RHCOS image, you could hack it, using a local web server to serve the RHCOS image and change the line with the image to something like this:

- name: OPENSHIFT_VERSIONS
        value: '{"4.8":{"rhcos_image":"http://<LOCAL-INSTALLER-WEBSEFVER_IP>/rhcos-48.84.202104252100-0-live.x86_64.iso","rhcos_version": "48.84.202104252100-0"}}'

Using this config map

apiVersion: v1
kind: ConfigMap
metadata:
  name: my-assisted-service-config
  namespace: assisted-installer
data:
  LOG_LEVEL: "debug"

Deploy Agent

---
apiVersion: agent-install.openshift.io/v1beta1
kind: AgentServiceConfig
metadata:
  namespace: assisted-installer
  name: agent
  annotations:
    unsupported.agent-install.openshift.io/assisted-service-configmap: "my-assisted-service-config"
spec:
  databaseStorage:
    accessModes:
      - ReadWriteOnce
    resources:
      requests:
        storage: 20Gi
  filesystemStorage:
    accessModes:
      - ReadWriteOnce
    resources:
      requests:
        storage: 20Gi
  osImages:
    - openshiftVersion: "4.8"
      version: ""
      url: "https://mirror.openshift.com/pub/openshift-v4/dependencies/rhcos/pre-release/latest-4.8/rhcos-live.x86_64.iso"
      rootFSUrl: "https://mirror.openshift.com/pub/openshift-v4/dependencies/rhcos/pre-release/latest-4.8/rhcos-live-rootfs.x86_64.img"

Older Release Notes (not used now but keep as a document reference in case you need it):

Patch HTTPS in case of error

 Information

With the new releases tag this step is not necessary but keep it just to get a workaround

If you have a problem with the HTTPS, there is a hack to avoid it, using the next patch:

  1. For the value RHCOS image -> Get the https in local, and put it in a web server (maybe the installer /var/www/html/rhcos-48.84.202104252100-0-live.x86_64.iso)

    value: '{"4.8":{"rhcos_image":"https://<ip-installer-vm>/rhcos-48.84.202104252100-0-live.x86_64.iso","rhcos_version": "48.84.202104252100-0"}}'

  2. After that, create the deployment oc apply -f 04-ai-subscription.yml and wait until pods are ready

  3. Now, we have to scale to 0 replicas the operator

    oc scale --replicas=0 deployment/assisted-service-operator

  4. Then use oc edit deployment/assisted-service to:

    • Change https to http in the base-url
    • Remove SERVE_HTTPS to false
    • None in Auth-type instead of local
    • HTTP in liveness and readiness

  5. And use oc edit route/assisted-service to:

    • remove tls section (2 lines)

  6. Now you should have the operator running:

    [root@qct-d14u11 sno-baremetal]# oc get pod
NAME                                         READY   STATUS             RESTARTS   AGE
assisted-service-f5d56977-ldds8              2/2     Running            0          4h2m
assisted-service-operator-84cd55f754-wg56l   1/1     Running            0          4h3m

    and if you’ve applied the patch just have the service:

    [root@qct-d14u11 sno-baremetal]# oc get pod
NAME                                         READY   STATUS             RESTARTS   AGE
assisted-service-f5d56977-ldds8              2/2     Running            0          4h2m

NAME                           	READY   STATUS	RESTARTS   AGE

ocp-metal-ui-54b96c4657-d6svz  	1/1 	Running   0      	18h
postgres-5fcc9c9d66-2vcd5      	1/1 	Running   0      	18h
scality-66556c48bf-5bdtp       	1/1 	Running   0      	18h

Finally, expose the service ocp-metal-ui so the application can be accessed via web.

oc expose svc ocp-metal-ui

This will create the ocp-metal-ui to access the AI web console: ocp-metal-ui-assisted-installer.apps.<cluster-name>.<domain>

You need to add (/etc/hosts) entries to be accessible from the external host (using port 80)

When you get access to the Web User Interface: http://ocp-metal-ui-assisted-installer.apps.qct11.karmalabs.com/clusters

You could see an error that says that it’s not possible to get information about clusters. It’s a bug and to solve it:

[root@qct-d14u11 sno-ai-hive-baremetal]# oc get svc
NAME TYPE CLUSTER-IP EXTERNAL-IP PORT(S) AGE
assisted-service LoadBalancer 172.30.85.231 <pending> 8090:31932/TCP 38m
controller-manager-metrics-service ClusterIP 172.30.227.249 <none> 8443/TCP 38m
ocp-metal-ui LoadBalancer 172.30.179.237 <pending> 80:31732/TCP 38m
postgres LoadBalancer 172.30.255.43 <pending> 5432:31168/TCP 38m

You have to edit the config map for the ocp-metal-ui to change the proxy port reference to this bold value (31932)

oc edit cm ocp-metal-ui
