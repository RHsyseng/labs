Scenario Description

The main goal of this document is to explain the deployment of a baremetal host over Assisted Installer. This lab uses sushy-tools to emulate the same situation which we could found using a real Baremetal host.

Requirements

  • Hub cluster deployed successfully
  • LocalStorage ready
  • Hive operator ready
  • Assisted Installer Operator ready

Install sushy-tools

 Information

If you’ve installed IPI baremetal with Kcli and virtual_protocol: Redfish -> you could omit this step because it is already installed.

This will create a service running in the port :8000 which will be the emulation for the Redfish protocol

dnf -y install pkgconf-pkg-config libvirt-devel gcc python3-libvirt python3 git
pip3 install sushy-tools
IP=$(ip -o addr show eth0 | head -1 | awk '{print $4}' | cut -d "/" -f 1 | head -1)
sed -i "s/127.0.0.1/$IP/" /etc/sushy.conf
# cp /root/bin/sushy.service  /usr/lib/systemd/system
# restorecon -Frv /usr/lib/systemd/system/sushy.service
systemctl enable --now sushy
sleep 20
ssh-keyscan -H {{ config_host }} >> /root/.ssh/known_hosts
echo -e "Host=*\nStrictHostKeyChecking=no\n" > /root/.ssh/config
python3 /root/bin/sushy.py

At this point make sure you have the next option by default:

SUSHY_EMULATOR_IGNORE_BOOT_DEVICE = True

Create the VM (host)

Create the image with uefi=true and start=false because we’re gonna use the sushy tools. If you’re using a SNO only have to use one of them.

kcli --debug create vm -P uefi_legacy=true -P start=false -P nets=['{"name":"bare-net", "mac":"ee:bb:aa:ee:1e:1a"}'] -P memory=38000 -P numcpus=8 -P disks=[180] agent1
kcli --debug create vm -P uefi_legacy=true -P start=false -P nets=['{"name":"bare-net", "mac":"ee:bb:aa:ee:1e:1b"}'] -P memory=38000 -P numcpus=8 -P disks=[180] agent2
kcli --debug create vm -P uefi_legacy=true -P start=false -P nets=['{"name":"bare-net", "mac":"ee:bb:aa:ee:1e:1c"}'] -P memory=38000 -P numcpus=8 -P disks=[180] agent3

sleep 2

virsh dumpxml agent1 | grep -i uuid
echo; echo; echo;
virsh dumpxml agent2 | grep -i uuid
echo; echo; echo;
virsh dumpxml agent3 | grep -i uuid

Then the idea is to get the UUID (from the head of the file) from the VM: To do that we’ve prepared the 04-create-new-vm.sh script

Modify the BareMetalHost YAML

Now, the idea is to prepare the BMH which we’re gonna create in order to wait until the VM is ready.

To do that, the operator is looking for the agent UUID and the MAC address. If both flags match with the information we’re gonna load, we will move forward next step.

Take a look to the MAC, the UUID at the end of the address and the discovery.iso HTTPD file server which we’ve created before

---
apiVersion: v1
kind: Secret
metadata:
  name: bmc-secret1
  namespace: assisted-installer
data:
  password: YWxrbm9wZmxlcgo=
  username: YW1vcmdhbnQK
type: Opaque

---
apiVersion: metal3.io/v1alpha1
kind: BareMetalHost
metadata:
  name: lab-agent1
  namespace: assisted-installer
  labels:
    infraenvs.agent-install.openshift.io: "lab-env"
  annotations:
    inspect.metal3.io: disabled
spec:
  online: true
  bmc:
    address: redfish-virtualmedia+http://INSTALLER_IP:8000/redfish/v1/Systems/AGENT1
    credentialsName: bmc-secret1
    disableCertificateVerification: true
  bootMACAddress: ee:bb:aa:ee:1e:1a
  automatedCleaningMode: disabled
