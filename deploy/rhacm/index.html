Install the RHACM upstream (only connected)

This is the official way to deploy the release upstream. Pretty simple, just creating the next manifests:

First thing is to create the namespace with oc apply -f 01-rhacm-ns.yml

apiVersion: v1
kind: Namespace
metadata:
  labels:
    openshift.io/cluster-monitoring: "true"
  name: open-cluster-management

Then you should create the operator group with oc apply -f 02-operatorGroup.yml

apiVersion: operators.coreos.com/v1
kind: OperatorGroup
metadata:
  name: advanced-cluster-management-operatorgroup
  namespace: open-cluster-management
spec:
  targetNamespaces:
    - open-cluster-management

Next thing is the subscription. Here, we have to define the channel with the correct release we’re gonna deploy and the source where the release could be found:

apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: advanced-cluster-management-subscription
  namespace: open-cluster-management
spec:
  channel: "release-2.2"
  name: advanced-cluster-management
  source: redhat-operators
  sourceNamespace: openshift-marketplace

Now you should wait a little until all the basic resources have been created in order to continue with the CR MultiClusterHub with oc apply -f 05-multiclousterhub-cr.yml:

apiVersion: operator.open-cluster-management.io/v1
kind: MultiClusterHub
metadata:
  name: multiclusterhub
  namespace: open-cluster-management
spec:
  imagePullSecret: ""

At this point we’ve to wait until all the resources have been created. It can take up to 10 minutes for the MultiClusterHub

You could verify using ../shared-utils/wait_for_acm.sh or maybe doing a basic check:

oc get mch -o=jsonpath='{.items[0].status.phase}'

Moreover, you should verify that the routes are available to access through the web console:

oc get routes
