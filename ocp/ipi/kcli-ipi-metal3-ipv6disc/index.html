Requirements:

This documents, starts from an installation ready with IPI Baremetal 4.8 - Disconnected / IPv6 but using virtual VM to simulate a real scenario.

In our case, we don’t need the lab mode so we’re just installing using the next lab.yml.

AUTOMATIC installation

With this three parameters, the installation will be automatic from scratch to finish:

  • lab: true
  • launch_steps: true
  • deploy_openshift: true

Edit your lab_ipv6.yml

lab: true
provisioning_enable: false
pool: default
disconnected: true
virtual_masters: true
launch_steps: true
deploy_openshift: true
version: ci
tag: 4.8.0-0.nightly-2021-06-01-014458
openshift_version: 4.8
cluster: lab
domain: alklabs.com
openshift_image: registry.ci.openshift.org/ocp/release:4.8.0-0.nightly-2021-06-01-014458
baremetal_cidr: 2620:52:0:1302::/64
baremetal_net: lab-net
provisioning_net: lab-prov
virtual_masters_memory: 16384
virtual_masters_numcpus: 8
virtual_workers_deploy: false
api_ip: 2620:52:0:1302::2
ingress_ip: 2620:52:0:1302::3
disk_size: 300
ipmi_user: jimi
ipmi_password: hendrix
disconnected_operators:
  - advanced-cluster-management
  - local-storage-operator
  - ocs-operator
  - performance-addon-operator

  - sriov-network-operator

  - aa:aa:aa:aa:bb:01

The OCP release will be mirrored to this environment:

version: ci
tag: 4.8.0-0.nightly-2021-06-01-014458
openshift_version: 4.8
cluster: lab
domain: alklabs.com
openshift_image: registry.ci.openshift.org/ocp/release:4.8.0-0.nightly-2021-06-01-014458

For sure, the most important is that automatically with the plan the release OCP will be mirrored

As you can see, you can also specify several disconnected operators which will be mirrored from OLM (official) to be available without an external connection:

NOTE: with this, you cannot use an specific build or bundle… It’s only for the official operators.

disconnected_operators:
  - advanced-cluster-management
  - local-storage-operator
  - ocs-operator
  - performance-addon-operator
  - ptp-operator
  - sriov-network-operator

To create the environment ipv6 disconnected with the OCP using VMs just launch it using:

kcli create plan --paramfile=lab-ipv6.yml test-ci-ipv6

Workflow snip

name: ocp_bm_ipi+IPv6Disc+ACM+Spoke+BMH
on:
  workflow_dispatch:
    inputs:
      OC_RELEASE:
        description: "ocp release: -> registry.ci.openshift.org/ocp/release:4.8.0-0.nightly-2021-04-30-154520 <-"
        required: true
      OC_SNAPSHOT:
        description: "ACM Snapshot -> 2.3.0-DOWNSTREAM-2021-06-13-16-46-23 <-"
        required: true
      OC_BUNDLE:
        description: "ACM Bundle -> v2.3.0-124 <-"
        required: true
      OC_DEPLOY_OCP:
        description: "Deploy OCP? [yes|no]"
        required: false
        default: "yes"
      OC_MIRROR:
        description: "Run Mirroring? [yes|no]"
        required: false
        default: "yes"
      OC_DEPLOY_ACM:
        description: "Deploy ACM with Hive and AI[yes|no]"
        required: false
        default: "yes"
      OC_DEPLOY_HUB_SPOKE_ACM:
        description: "Deploy Hub and Spoke over ACM [yes|no]"
        required: false
        default: "yes"
      OC_MASTER_SPOKE:
        description: "Number of Master spoke (SNO/Multinode) -> [1|3] <-"
        required: false
        default: "1"
      OC_CLUSTER_NAME:
        description: "Clusters Name -> myCluster <-"
        required: false
        default: "test-ci"
      RUNNER:
        description: "Github Action Runner Name"
        required: true

env:
  DEPLOY_OCP_DIR: lab-kcli-ipi-baremetal
  LOCALSTORAGE_DIR: deploy-LocalStorage-operator
  SPOKE_ACM_DIR: deploy-hub-spoke-over-acm
  SHARED_DIR: shared-utils
  ACM_DIR: deploy-rhacm
  BMHOST_DIR: deploy-BMHost-virtual-over-acm

jobs:
  requirements:
    runs-on: ${{github.event.inputs.RUNNER}} #tag with your runner tag

    steps:
      - name: Install kcli and skopeo
        run: |
          dnf -y copr enable karmab/kcli
          dnf clean all ; dnf -y install kcli skopeo git
          kcli create pool -p /var/lib/libvirt/images default
          sudo setfacl -m u:$(id -un):rwx /var/lib/libvirt/images
          kcli download image centos8          

      - uses: actions/checkout@v2 # create a checkout (pull repository) in the runner working_dir (usually _work)
      - run: git pull origin ${GITHUB_REF##*/}

      - name: Install clients
        run: |
          cd ${{env.SHARED_DIR}}
          ./get_clients.sh          

      - name: Setup the main pull-secret
        run: |
          cd ${{env.DEPLOY_OCP_DIR}}
          echo ${{secrets.AMORGANT_PULL_SECRET}} | tr -d [:space:] > openshift_pull.json          

  deploy-ocp-ipv6-disc:
    needs: requirements # create dependencies between jobs
    runs-on: ${{github.event.inputs.RUNNER}}
    if: github.event.inputs.OC_DEPLOY_OCP == 'yes'
    steps:
      - name: Deploy using kcli baremetal plan
        run: |
          cd ${{env.DEPLOY_OCP_DIR}}
          t=$(echo ${{github.event.inputs.OC_RELEASE}} | awk -F: '{print $2}')
          kcli create plan --paramfile=lab_ipv6.yml -P tag=$t -P openshift_image=${{github.event.inputs.OC_RELEASE}} ${{github.event.inputs.OC_CLUSTER_NAME}}          

  verify:
    needs: deploy-ocp-ipv6-disc
    runs-on: ${{github.event.inputs.RUNNER}}
    if: github.event.inputs.OC_DEPLOY_OCP == 'yes' && !failure()
    steps:
      - name: verify installation success ipv6 disconnected
        run: |
          cd ${{env.SHARED_DIR}}
          ./wait_until_ocp_bm_ipv6_ready.sh ${{github.event.inputs.OC_CLUSTER_NAME}} 3 ${{env.DEPLOY_OCP_DIR}}
